Знание
Зачем это учить?

В любой команде, даже студенческой, по разработке игр важно умение работать с git, с практически любым движком. Так вы сможете работать в одном проекте одновременно, каждый над своей задачей.

Как это учить?

Приходить на пары, смотреть записи.

 

Задание

Сдать SSH-ссылку на GitHub репозиторий.


Репозиторий должен содержать в себе минимум 2 ветки (мастер и ветка new_branch)
Ветка new_branch должн быть вмерджена в ветку master
Должен быть решён конфликт слияния при слиянии ветки new_branch в ветку master
Репозиторий должен содержать файлы .gitignores и .gitattributes (описать содержимое файлов)
Репозиторий должен содержать LFS-файлы

 

Критерии оценки работы:
50 баллов — репозиторий сделан на GitHub. Ссылка для скачивания валидна.
70 баллов — репозиторий сделан на GitHub. Содержит всю историю изменений до момента мерджа веток. Ссылка валидна.
100 баллов — репозиторий сделан на GitHub. Содержит всю историю изменений до момента мерджа веток. Созданы файлы .gitattributes и .gitignore, с описанными ниже правилами. Ссылка валидна.

 

Репозиторий должен содержать следующую историю изменений:
добавляется файл logo.png
добавляется файл .gitignore, в котором помечается игнорирование файлов с расширением .obj и игнорирование всех файлов в дочерней папке от папки inside_folder с расширением .log
добавляется файл .gitattributes с пометкой файлов .png как LFS файл
добавляется новый коммит с пустым файлом inside_text.txt
в мастер вливается new_branch и решается конфликт слияния
добавляем текст в файл hello_text.txt в ветке master new_branch
добавляем текст в файл hello_text.txt в ветке master
инициализация репозитория, первый коммит

 

Иерархическое дерево:
root
 hello_text.txt
 inside_folder
   inside_text.txt
   logo.png
